Frontend techniques​

There are many CSS frameworks out there. I listed some of them that I used earlier but I decided to write this portfolio with a BEM approach (block, element, modifier) which helps me to name CSS classes. At the same time it helps me with writing clean and transparent HTML markup. Also, I used the CSS Flexbox Layout Module, which makes it easier to design a flexible responsive layout structure without using float or positioning. Also its Browser Support is really good.

Java Script

  • Vanilla JS

  • React (Context API, Redux), React Native

  • Usage of JS libraries​: Jquery, Highcharts, Scrollmagic, Data-fns

My previous experience with React would easier setup of this project. Despite that I opted for pure native JavaScript and an object-oriented way of codding. In addition to much faster loading and execution, modern JavaScript also has excellent browser support. To write highly organized JavaScript I usually structure my code in a few steps: 1) Selecting elements from the DOM 2) Event handling 3) Defining functionality. This type of organization I used for a few features (mobile menu, sticky header, revealing boxes on a scroll, modal window) that can be found on this project.

Package and task managers​

  • Webpack

  • Node.js

  • Gulp

  • Grunt

Two crucial things, from my point of view, during building a site or an app are automation and organization. Automation is when we force a computer to execute some task for us. The organization is clearly written code (many separate files which are related to a certain part of a website ) and just one bundled file (css & js) which browser will load quickly to satisfy high performances. To accomplish all of that here I decided to use Webpack, which is really great tool. With Webpack I made different setup for the development and production environment.
